<h1 align="center">Como os sites funcionam?</h1>
<br>

| Perguntas-chave | Notas principais |
|---|---|
| Quais são os 3 tipos de arquivo que um site usa? | HTML, CSS e JavaScript |
| O que o HTML faz? | Define o **conteúdo** da página: textos, imagens, botões, links |
| O que o CSS faz? | Define a **aparência**: cores, fontes, bordas, layout |
| O que o JavaScript faz? | Define a **funcionalidade**: interatividade, ações do usuário |
| O que é o Chrome DevTools? | Ferramenta do Chrome pra inspecionar e editar o código de qualquer site em tempo real |
| Por que editar no DevTools não salva? | Você edita só a cópia local no navegador, ao atualizar, o servidor envia os arquivos originais de novo |

---

> 💡 **Conceito central:** Todo site é formado por três tipos de arquivo com funções completamente diferentes: HTML (conteúdo), CSS (aparência) e JavaScript (funcionalidade).

Quando você acessa um site, o servidor envia para o seu navegador três tipos de arquivos. Cada um tem uma responsabilidade distinta, e juntos eles formam tudo que você vê e usa na web. Para entender o papel de cada um, a professora usa uma analogia com uma casa:

- **HTML** → os tijolos e a estrutura da casa
- **CSS** → a pintura, decoração e formato das janelas
- **JavaScript** → a instalação elétrica, os eletrodomésticos, tudo que faz a casa funcionar de verdade

---

### HTML: o conteúdo

> 💡 **Conceito central:** HTML define *o que existe* na página, não como ela parece.

O HTML (HyperText Markup Language) é a base de qualquer site. Ele define o conteúdo: textos, imagens, botões, links e qualquer outro elemento visível. Sem HTML não existe nada, é a estrutura crua da página. 

---

### CSS: a aparência

> 💡 **Conceito central:** CSS não cria elementos novos — ele estiliza os elementos que o HTML criou.

O CSS (Cascading Style Sheets) transforma a aparência dos elementos HTML. Ele define cor de fundo, formato dos botões, tamanho de fonte, espaçamentos e qualquer detalhe visual. 

---

### JavaScript: a funcionalidade

> 💡 **Conceito central:** JavaScript transforma um site estático em algo que o usuário pode realmente usar.

O JavaScript é o que dá vida ao site. Ele permite que o usuário interaja com a página: enviar um e-mail no Gmail, publicar uma foto no Instagram, pesquisar no Google. Sem JavaScript, os sites seriam apenas páginas de leitura, bonitas mas sem nenhuma interatividade.

---

### Chrome DevTools: inspecionando sites ao vivo

> 💡 **Conceito central:** O DevTools permite ver e editar o código de qualquer site diretamente no navegador, mas as alterações são temporárias.

Para abrir o DevTools: clique com o botão direito em qualquer elemento da página → **Inspecionar**. O Chrome vai destacar automaticamente o trecho de código HTML responsável pelo elemento clicado. Você pode dar duplo clique em qualquer texto e editá-lo na hora. Por exemplo, mudar o botão "Google Search" para qualquer outro texto.

**Importante:** essa edição acontece só na sua cópia local. Ao atualizar a página, o navegador busca os arquivos originais no servidor novamente e tudo volta ao normal. Você não está alterando o site de verdade.
